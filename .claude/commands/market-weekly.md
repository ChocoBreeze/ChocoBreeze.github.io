# /market-weekly — 미국 증시 위클리 리포트 블로그 포스트 생성

사용자가 제공한 위클리 리포트 내용을 블로그 포스트로 저장한다.

## 입력 형식

$ARGUMENTS 에는 아래 정보가 포함된다:

- **본문 내용** (필수): 사용자가 작성한 위클리 리포트 전문 (마크다운 형식)
- **날짜** (선택): "어제", "오늘", 또는 `YYYY-MM-DD` 형식. 생략 시 오늘(Asia/Seoul 기준)

## 처리 절차

### 1. 날짜 결정
- 입력에 날짜가 명시된 경우 그 날짜 사용
- "어제"이면 오늘 날짜에서 하루 뺀 날짜 사용
- 생략 시 오늘 날짜(Asia/Seoul, UTC+9) 사용
- 형식: `YYYY-MM-DD`

### 2. 파일 스캐폴드

아래 명령어로 파일을 생성한다:

```bash
npm run new:post -- --type market-weekly --date {YYYY-MM-DD}
```

출력에서 생성된 파일 경로를 확인한다.

### 3. 파일 내용 교체

스캐폴드된 파일의 **프론트매터는 그대로 유지**하고, 프론트매터 아래 템플릿 내용을 사용자가 제공한 본문으로 교체한다.

최종 파일 구조:
```
---
(스캐폴드된 프론트매터 그대로)
---

(사용자가 제공한 본문 전문)
```

### 4. 검증

파일 생성 후 반드시 실행:
```bash
npm run check:content
```

검증 통과 시 완료를 알린다. 실패 시 오류를 수정하고 재검증한다.

## 작성 지침

- 본문 내용을 임의로 수정하거나 추가하지 않는다. 사용자가 제공한 내용을 그대로 사용한다.
- 프론트매터(title, pubDate, categories, slug)는 스캐폴드 결과를 그대로 유지한다.
- 날짜가 명시되지 않은 경우 반드시 사용자에게 확인하지 않고 오늘 날짜를 사용한다.
