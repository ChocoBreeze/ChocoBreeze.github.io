# /leetcode — LeetCode 풀이 블로그 포스트 생성

사용자가 제공한 LeetCode 문제와 풀이를 블로그 포스트로 작성한다.

## 입력 형식

$ARGUMENTS 에는 아래 정보가 포함된다:

- **문제 URL** (선택): `https://leetcode.com/problems/...` 또는 `today` / 생략. URL을 주지 않거나 "today"라고 입력하면 `https://lctp.fly.dev/` 를 curl로 호출해 오늘의 문제 URL을 자동으로 가져온다.
- **내 아이디어** (선택): 코드 작성 전에 떠올린 접근법을 텍스트로 설명. `### 내 아이디어` 또는 `내 아이디어:` 형태로 구분한다. 생략 시 코드를 보고 아이디어를 역으로 추론해 작성한다.
- **내 코드** (필수): 사용자가 직접 작성한 C++ 풀이. `내 코드:` 또는 첫 번째 코드블록으로 구분한다.
- **풀이 1, 풀이 2, ...** (선택): 내 코드 이외의 다른 풀이. `### 풀이 1`, `### 풀이 2` 같은 제목으로 구분하거나 각 풀이 앞에 간단한 설명을 붙여 나열한다.
- **날짜** (선택): "어제", "오늘", 또는 `YYYY-MM-DD` 형식. 생략 시 오늘(Asia/Seoul 기준)

## 처리 절차

### 1. 날짜 결정
- 입력에 날짜가 명시된 경우 그 날짜 사용
- "어제"이면 오늘 날짜에서 하루 뺀 날짜 사용
- 생략 시 오늘 날짜(Asia/Seoul, UTC+9) 사용
- 형식: `YYYY-MM-DD`

### 2. 문제 URL 확정

- 입력에 `https://leetcode.com/problems/...` URL이 있으면 그대로 사용한다.
- URL이 없거나 "today"이면 아래 명령으로 오늘의 문제 URL을 가져온다:
  ```bash
  curl -sL https://lctp.fly.dev/
  ```
  반환값이 곧 문제 URL이다.

### 3. 문제 정보 파악
URL에서 문제 번호와 이름을 추출하고, 아래 정보를 결정한다:
- **제목**: `{번호}. {영문 제목}` (LeetCode 표기 그대로)
- **난이도**: Easy / Medium / Hard
- **토픽**: LeetCode 문제 태그 기준 (예: Array, Greedy, Sorting, Dynamic Programming 등)

### 4. 파일 생성

**파일 경로:**
```
src/content/blog/Problem Solving/{날짜}-{번호}. {영문 제목}.md
```

**프론트매터:**
```yaml
---
layout: post
title: {번호}. {영문 제목}
date: {날짜} 20:00:00 +0900
tags: [Problem_Solving]
categories: Problem_Solving
difficulty: {Easy|Medium|Hard}
topics: [{토픽1}, {토픽2}, ...]
---
```

**본문 구성:**


### 5. 검증

파일 생성 후 반드시 실행:
```bash
npm run check:content
```

검증 통과 시 완료를 알린다. 실패 시 오류를 수정하고 재검증한다.

## 작성 지침

- 설명은 **한국어**로 작성한다.
- 코드 블록은 `cpp` 언어 태그를 사용한다.
- 수식은 KaTeX 문법(`$...$`, `$$...$$`)을 사용한다.
- 비교표가 있으면 마크다운 테이블로 정리한다.
- 예시가 있으면 단계별로 추적해서 보여준다.
- 내 코드와 추가 풀이가 동일한 알고리즘이면, 차이점(가독성, 간결성 등)을 솔직하게 설명한다.
- 주석이나 불필요한 설명 없이 내용에 집중한다.
- 추가 풀이가 없으면 `## 풀이 비교` 섹션을 생략한다.
