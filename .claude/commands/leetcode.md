# /leetcode — LeetCode 풀이 블로그 포스트 생성

사용자가 제공한 LeetCode 문제와 풀이를 블로그 포스트로 작성한다.

## 입력 형식

$ARGUMENTS 에는 아래 정보가 포함된다:

- **문제 URL** (필수): `https://leetcode.com/problems/...`
- **내 코드** (필수): 사용자가 직접 작성한 C++ 풀이
- **더 좋은 풀이** (선택): 내 코드보다 나은 풀이가 있을 경우 추가로 제공. 여러 개면 `### 풀이 1`, `### 풀이 2` 같은 제목으로 구분하거나, 각 풀이 앞에 간단한 설명(예: "O(n) 카운팅 풀이", "투 포인터 풀이")을 붙여서 나열한다.
- **날짜** (선택): "어제", "오늘", 또는 `YYYY-MM-DD` 형식. 생략 시 오늘(Asia/Seoul 기준)

## 처리 절차

### 1. 날짜 결정
- 입력에 날짜가 명시된 경우 그 날짜 사용
- "어제"이면 오늘 날짜에서 하루 뺀 날짜 사용
- 생략 시 오늘 날짜(Asia/Seoul, UTC+9) 사용
- 형식: `YYYY-MM-DD`

### 2. 문제 정보 파악
URL에서 문제 번호와 이름을 추출하고, 아래 정보를 결정한다:
- **제목**: `{번호}. {영문 제목}` (LeetCode 표기 그대로)
- **난이도**: Easy / Medium / Hard
- **토픽**: LeetCode 문제 태그 기준 (예: Array, Greedy, Sorting, Dynamic Programming 등)

### 3. 파일 생성

**파일 경로:**
```
src/content/blog/Problem Solving/{날짜}-{번호}. {영문 제목}.md
```

**프론트매터:**
```yaml
---
layout: post
title: {번호}. {영문 제목}
date: {날짜} 20:00:00 +0900
tags: [Problem_Solving]
categories: Problem_Solving
difficulty: {Easy|Medium|Hard}
topics: [{토픽1}, {토픽2}, ...]
---
```

**본문 구성:**

```markdown
[문제 링크]({URL})

<br/>

## 문제 설명

{문제를 한국어로 설명한다. 다음 항목을 포함:}
- 문제에서 주어지는 것 (입력, 조건)
- 수행 가능한 연산
- 구해야 하는 것 (목표)
- 핵심 관찰 (풀이 방향을 잡는 데 도움이 되는 수학적/논리적 사실)

<br/>

## 내 코드 분석

{사용자 코드 코드블록}

- **Time Complexity:** $O(...)$
- **Space Complexity:** $O(...)$

### 동작 원리

{단계별로 코드의 논리를 설명한다. 핵심 invariant, 왜 이 접근이 맞는지 포함.}

<br/>

## 최적 풀이

{더 좋은 풀이가 1개인 경우: 코드블록 + 분석}
{더 좋은 풀이가 여러 개인 경우: ### Approach 1, ### Approach 2 … 로 나눠서 각각 코드블록 + 분석 작성. 마지막에 풀이들을 비교하는 표 추가.}
{더 좋은 풀이가 없는 경우: "내 코드가 이미 최적이다" 또는 같은 접근의 더 간결한 버전 제시}

- **Time Complexity:** $O(...)$
- **Space Complexity:** $O(...)$

### 핵심 아이디어

{왜 이 풀이가 더 나은지, 내 코드와의 차이점을 비교표로 정리}

| 항목 | 내 코드 | 최적 풀이 |
|------|---------|-----------|
| 시간 복잡도 | ... | ... |
| 공간 복잡도 | ... | ... |
| ... | ... | ... |

<br/>
```

### 4. 검증

파일 생성 후 반드시 실행:
```bash
npm run check:content
```

검증 통과 시 완료를 알린다. 실패 시 오류를 수정하고 재검증한다.

## 작성 지침

- 설명은 **한국어**로 작성한다.
- 코드 블록은 `cpp` 언어 태그를 사용한다.
- 수식은 KaTeX 문법(`$...$`, `$$...$$`)을 사용한다.
- 비교표가 있으면 마크다운 테이블로 정리한다.
- 예시가 있으면 단계별로 추적해서 보여준다.
- 내 코드와 최적 풀이가 동일한 알고리즘이면, 차이점(가독성, 간결성 등)을 솔직하게 설명한다.
- 주석이나 불필요한 설명 없이 내용에 집중한다.
