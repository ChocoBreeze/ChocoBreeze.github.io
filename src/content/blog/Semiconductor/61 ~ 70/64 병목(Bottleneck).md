## 1) 한 문장 정의

> **병목은 전체 시스템에서 가장 느려서, 나머지 빠른 부품들까지 함께 기다리게 만드는 ‘가장 좁은 부분’이야.**

---

## 2) 비유

병목이라는 말은 원래 **병 입구가 좁다**는 데서 나온 말이야.

큰 물통에서 물을 엄청 많이 부어도 병 입구가 좁으면 물이 한꺼번에 못 나가고 막히지?

반도체도 똑같아.

예를 들면:

* GPU는 엄청 빠른데
* HBM이 데이터를 늦게 주면

→ GPU가 기다림

또는:

* HBM은 충분한데
* 패키징(CoWoS)이 부족하면

→ GPU 자체를 많이 못 만듦

즉, **가장 느리거나 부족한 부분이 전체 속도를 결정**하는 거야.

---

## 3) 실제 예시

반도체 산업에서 병목은 엄청 자주 나와.

### 예시 1: AI GPU + HBM

* GPU 성능은 엄청 높은데
* HBM 공급이 부족하면

→ GPU 생산이 병목

### 예시 2: HBM + CoWoS

* HBM 메모리는 충분한데
* 고급 패키징(CoWoS) 용량이 부족하면

→ 최종 제품 출하가 병목

### 예시 3: CPU + DRAM

* CPU는 빠른데
* 메모리 지연시간이 길면

→ 메모리 병목

### 예시 4: SSD + PCIe

* SSD 자체는 빠른데
* 연결 버스가 느리면

→ PCIe 병목

## 4) 시각적 이미지 설명

머릿속에 이렇게 그려봐:

```text
넓은 길 → 넓은 길 → 좁은 길 → 넓은 길
```

앞뒤 길이 아무리 넓어도 가운데 **좁은 길 하나** 때문에 전체 차가 밀리게 돼.

반도체 시스템도 똑같아.

* CPU
* GPU
* 메모리
* 버스
* 패키징
* 전력
* 냉각

이 중 어디 하나가 좁으면 전체 성능이 거기서 막혀버려.

---

## 5) 마켓 관점

병목을 찾는 건 산업에서 진짜 중요해.

왜냐면:

✔ 돈을 어디에 투자해야 할지 결정되기 때문  
✔ “뭐가 부족해서 성능/생산이 안 나오는지” 알아야 하기 때문

예를 들어 AI 시대에는 병목이 이렇게 계속 바뀌었어:

* 처음엔 **GPU 부족**
* 그다음은 **HBM 부족**
* 또 그다음은 **CoWoS 부족**
* 앞으로는 **전력/냉각/데이터센터 인프라 부족**도 병목이 될 수 있어

즉, **산업을 보는 눈 = 지금 어디가 병목인지 보는 눈**이라고 해도 될 정도야.
