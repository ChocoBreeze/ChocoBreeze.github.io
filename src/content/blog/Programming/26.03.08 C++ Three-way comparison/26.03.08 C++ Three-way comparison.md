`<=>`는 두 객체의 비교 결과를 카테고리 타입으로 반환해, 비교 연산자를 더 일관되게 구성하도록 돕는 C++20 기능입니다.

## 왜 필요한가

* `==`, `<`, `<=`, `>`, `>=`를 따로 구현할 때 생기는 중복을 줄일 수 있다.
* 비교 의미를 `strong`, `weak`, `partial` 같은 타입으로 드러낼 수 있다.
* 정렬과 연관 컨테이너에서 비교 규칙을 더 명확하게 유지할 수 있다.

## 핵심 개념

### 비교 카테고리

* `std::strong_ordering`: 강한 순서
* `std::weak_ordering`: 동치 클래스가 있을 수 있는 약한 순서
* `std::partial_ordering`: 일부 값쌍은 비교 불가

### 기본 구현

```cpp
#include <compare>
#include <string>

struct User {
    int id;
    std::string name;

    auto operator<=>(const User&) const = default;
};
```

### 수동 구현

```cpp
#include <compare>
#include <string>

struct Product {
    std::string sku;
    double price;

    std::strong_ordering operator<=>(const Product& rhs) const {
        if (auto c = sku <=> rhs.sku; c != 0) {
            return c;
        }
        return std::strong_ordering::equal;
    }

    bool operator==(const Product& rhs) const {
        return sku == rhs.sku;
    }
};
```

## 실전에서 볼 포인트

* 부동소수점 멤버가 있으면 `partial_ordering`이 될 수 있다.
* `= default`는 멤버 선언 순서대로 비교하므로 정렬 기준을 꼭 확인해야 한다.
* `<=>`를 수동 구현하면 `operator==`의 의미도 같이 맞춰야 한다.

## 추천 연습

1. `Point` 타입에 `operator<=> = default`를 적용한다.
2. 특정 키만 비교하는 수동 구현을 작성한다.
3. `NaN`이 포함된 경우를 따로 테스트한다.
4. `std::sort`, `std::set`에서 비교 일관성을 검증한다.

## 체크리스트

* [ ] 내 타입의 비교가 강한/약한/부분 순서 중 무엇인지 설명할 수 있다.
* [ ] `= default`와 수동 구현을 언제 선택할지 안다.
* [ ] `operator==`와 `<=>`의 의미가 모순되지 않는다.
* [ ] 경계값 테스트를 분리해 검증했다.
