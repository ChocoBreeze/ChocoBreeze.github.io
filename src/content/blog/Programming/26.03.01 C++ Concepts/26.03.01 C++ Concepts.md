`Concepts`는 템플릿 인자가 만족해야 할 조건을 이름 붙여 표현하는 C++20 기능입니다.

## 왜 필요한가

* 템플릿이 기대하는 타입 조건을 코드에 직접 드러낼 수 있다.
* SFINAE보다 읽기 쉬운 컴파일 오류를 얻기 쉽다.
* 제약이 더 구체적인 오버로드를 우선 선택하게 만들 수 있다.

## 핵심 문법

### `concept` 정의

```cpp
#include <concepts>

template <typename T>
concept Integral = std::integral<T>;
```

### `requires`로 제약 걸기

```cpp
template <typename T>
requires Integral<T>
T add(T a, T b) {
    return a + b;
}
```

### 축약 템플릿

```cpp
Integral auto add2(Integral auto a, Integral auto b) {
    return a + b;
}
```

## 실전에서 볼 포인트

* `std::integral`, `std::floating_point`, `std::same_as` 같은 표준 concept를 먼저 익히는 편이 좋다.
* 제약은 가능한 한 최소한으로 적어야 과도한 제한을 피할 수 있다.
* `requires` 표현식은 멤버 함수 존재, 반환 타입, `noexcept` 여부까지 기술할 수 있다.

## 추천 연습

1. 정수만 받는 함수와 실수만 받는 함수를 각각 만든다.
2. 특정 멤버 함수가 있는 타입만 받는 concept를 작성한다.
3. 제약 실패 시 컴파일 오류 메시지가 어떻게 달라지는지 확인한다.
