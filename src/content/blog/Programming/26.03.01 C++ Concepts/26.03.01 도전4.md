# 도전4 - 제약 기반 mini ranges 파이프라인

## 목표

`std::ranges` + `concept`를 결합해 "필터 -> 변환 -> 수집" 파이프라인 유틸리티를 구현합니다.

## 요구 사항

- 함수 템플릿 `pipe_collect`를 작성합니다.
- 입력은 `input_range`만 허용합니다.
- `pred`는 `std::predicate` 제약을 만족해야 합니다.
- `proj`는 호출 가능해야 하며 결과 타입을 추론할 수 있어야 합니다.
- 출력 컨테이너는 `push_back` 가능한 타입만 허용합니다.

## 제약 조건

- SFINAE 대신 `requires` 또는 named concept를 사용합니다.
- 런타임 `if`로 타입 분기하지 않습니다.
- 실패 케이스 2개를 주석으로 남깁니다.

## 검증 체크리스트

- `std::vector<int>` 입력으로 홀수만 골라 제곱해서 수집되는가?
- `std::list<int>` 입력도 동일 로직으로 동작하는가?
- `push_back`이 없는 출력 타입에서 컴파일 에러가 나는가?
