[문제 링크](https://leetcode.com/problems/find-the-safest-path-in-a-grid/)

<br/>

## 문제 설명

`n × n` 이진 그리드 `grid`가 주어진다. `grid[i][j] == 1`이면 도둑(thief)이 있는 칸이고, `0`이면 빈 칸이다.

- 시작점: `(0, 0)`, 도착점: `(n-1, n-1)`
- 상하좌우 인접 칸으로 이동 가능

**경로의 safeness factor**는 경로 위 모든 칸에서 가장 가까운 도둑까지의 맨해튼 거리 중 **최솟값**으로 정의된다.

시작점에서 도착점까지 가는 경로 중 **safeness factor가 최대인 값**을 반환하라.

### 핵심 관찰

- 각 칸의 safeness는 "가장 가까운 도둑까지의 거리"이므로, 모든 도둑에서 동시에 BFS를 돌리면 $O(n^2)$에 전체 거리 테이블을 구할 수 있다.
- 경로의 safeness factor는 경로 위 칸들의 safeness 중 최솟값이므로, 이를 최대화하는 문제는 **최대-최소 경로 문제**다.
- Dijkstra와 유사하게 우선순위 큐로 "현재까지의 최소 safeness가 가장 큰 경로"를 먼저 탐색하면 최적해를 찾을 수 있다.

<br/>

## 내 코드 분석

```cpp
class Solution {
public:
    int maximumSafenessFactor(vector<vector<int>>& grid) {
        int n = grid.size();

        vector<int> dx = {-1, 1, 0, 0};
        vector<int> dy = {0, 0, -1, 1};

        // 1. 각 칸에서 가장 가까운 도둑까지의 거리 계산
        vector<vector<int>> dist(n, vector<int>(n, -1));
        queue<pair<int, int>> q;

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 1) {
                    dist[i][j] = 0;
                    q.push({i, j});
                }
            }
        }

        while (!q.empty()) {
            auto [x, y] = q.front();
            q.pop();

            for (int dir = 0; dir < 4; dir++) {
                int nx = x + dx[dir];
                int ny = y + dy[dir];

                if (nx < 0 || ny < 0 || nx >= n || ny >= n) continue;
                if (dist[nx][ny] != -1) continue;

                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }

        // 2. priority_queue로 safeness factor가 큰 경로부터 탐색
        priority_queue<pair<int, pair<int, int>>> pq;
        vector<vector<bool>> visited(n, vector<bool>(n, false));

        pq.push({dist[0][0], {0, 0}});

        while (!pq.empty()) {
            auto [safe, pos] = pq.top();
            pq.pop();

            auto [x, y] = pos;

            if (visited[x][y]) continue;
            visited[x][y] = true;

            if (x == n - 1 && y == n - 1) {
                return safe;
            }

            for (int dir = 0; dir < 4; dir++) {
                int nx = x + dx[dir];
                int ny = y + dy[dir];

                if (nx < 0 || ny < 0 || nx >= n || ny >= n) continue;
                if (visited[nx][ny]) continue;

                int nextSafe = min(safe, dist[nx][ny]);
                pq.push({nextSafe, {nx, ny}});
            }
        }

        return 0;
    }
};
```

- **Time Complexity:** $O(n^2 \log n)$ — 멀티소스 BFS $O(n^2)$ + 우선순위 큐 탐색 $O(n^2 \log n)$
- **Space Complexity:** $O(n^2)$

### 동작 원리

**1단계: 멀티소스 BFS로 거리 테이블 구성**

모든 도둑 위치를 초기 큐에 넣고(거리 0) 동시에 BFS를 수행한다. 각 칸 `dist[i][j]`에는 가장 가까운 도둑까지의 거리가 채워진다. 도둑이 한 곳이 아닌 여러 곳에서 동시에 퍼져나가므로 $O(n^2)$에 처리된다.

**2단계: 우선순위 큐로 최대-최소 경로 탐색**

Dijkstra와 유사한 구조다. 큐에는 `{현재까지의 최소 safeness, (x, y)}`를 넣는다. 항상 **safeness가 가장 큰 경로**를 먼저 꺼내 탐색하므로, `(n-1, n-1)`에 처음 도달했을 때의 safeness가 곧 최적값이다.

이웃 칸으로 이동할 때마다 `nextSafe = min(safe, dist[nx][ny])`로 경로의 병목(가장 취약한 칸)을 누적한다.

## 최적 풀이

### Approach 1: BFS + Binary Search

```cpp
class Solution {
public:

    int maximumSafenessFactor(vector<vector<int>>& grid) {
        int n = grid.size();

        queue<pair<int, int>> multiSourceQueue;
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 1) {
                    multiSourceQueue.push({i, j});
                    grid[i][j] = 0;
                } else {
                    grid[i][j] = -1;
                }
            }
        }

        while (!multiSourceQueue.empty()) {
            int size = multiSourceQueue.size();
            while (size-- > 0) {
                auto curr = multiSourceQueue.front();
                multiSourceQueue.pop();
                for (auto& d : dir) {
                    int di = curr.first + d[0];
                    int dj = curr.second + d[1];
                    int val = grid[curr.first][curr.second];
                    if (isValidCell(grid, di, dj) && grid[di][dj] == -1) {
                        grid[di][dj] = val + 1;
                        multiSourceQueue.push({di, dj});
                    }
                }
            }
        }

        int start = 0, end = 0, res = -1;
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
                end = max(end, grid[i][j]);

        while (start <= end) {
            int mid = start + (end - start) / 2;
            if (isValidSafeness(grid, mid)) {
                res = mid;
                start = mid + 1;
            } else {
                end = mid - 1;
            }
        }

        return res;
    }

private:

    vector<vector<int>> dir = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};

    bool isValidCell(vector<vector<int>>& grid, int i, int j) {
        int n = grid.size();
        return i >= 0 && j >= 0 && i < n && j < n;
    }

    bool isValidSafeness(vector<vector<int>>& grid, int minSafeness) {
        int n = grid.size();
        if (grid[0][0] < minSafeness || grid[n-1][n-1] < minSafeness) return false;

        queue<pair<int, int>> traversalQueue;
        vector<vector<bool>> visited(n, vector<bool>(n, false));
        traversalQueue.push({0, 0});
        visited[0][0] = true;

        while (!traversalQueue.empty()) {
            auto curr = traversalQueue.front();
            traversalQueue.pop();
            if (curr.first == n-1 && curr.second == n-1) return true;
            for (auto& d : dir) {
                int di = curr.first + d[0];
                int dj = curr.second + d[1];
                if (isValidCell(grid, di, dj) && !visited[di][dj] && grid[di][dj] >= minSafeness) {
                    visited[di][dj] = true;
                    traversalQueue.push({di, dj});
                }
            }
        }
        return false;
    }
};
```

- **Time Complexity:** $O(n^2 \log n)$ — BFS $O(n^2)$ × 이진 탐색 $O(\log n)$
- **Space Complexity:** $O(n^2)$

#### 핵심 아이디어

"safeness ≥ k인 칸만 지나서 도착점에 도달 가능한가?"라는 결정 문제는 단순 BFS로 $O(n^2)$에 판별할 수 있다. 이 결정 문제에 대한 답이 **단조(monotone)** 하므로(k가 커질수록 도달 불가능해짐) 이진 탐색을 적용해 최댓값을 $O(\log n)$번의 BFS로 찾는다.

### Approach 2: BFS + Greedy (Priority Queue)

```cpp
class Solution {
public:

    int maximumSafenessFactor(vector<vector<int>>& grid) {
        int n = grid.size();
        queue<pair<int, int>> multiSourceQueue;

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 1) {
                    multiSourceQueue.push({i, j});
                    grid[i][j] = 0;
                } else {
                    grid[i][j] = -1;
                }
            }
        }

        while (!multiSourceQueue.empty()) {
            int size = multiSourceQueue.size();
            while (size-- > 0) {
                auto curr = multiSourceQueue.front();
                multiSourceQueue.pop();
                for (auto& d : dir) {
                    int di = curr.first + d[0];
                    int dj = curr.second + d[1];
                    int val = grid[curr.first][curr.second];
                    if (isValidCell(grid, di, dj) && grid[di][dj] == -1) {
                        grid[di][dj] = val + 1;
                        multiSourceQueue.push({di, dj});
                    }
                }
            }
        }

        priority_queue<vector<int>> pq;
        pq.push({grid[0][0], 0, 0});
        grid[0][0] = -1;

        while (!pq.empty()) {
            auto curr = pq.top();
            pq.pop();

            if (curr[1] == n-1 && curr[2] == n-1) return curr[0];

            for (auto& d : dir) {
                int di = d[0] + curr[1];
                int dj = d[1] + curr[2];
                if (isValidCell(grid, di, dj) && grid[di][dj] != -1) {
                    pq.push({min(curr[0], grid[di][dj]), di, dj});
                    grid[di][dj] = -1;
                }
            }
        }

        return -1;
    }

private:

    vector<vector<int>> dir = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};

    bool isValidCell(vector<vector<int>>& mat, int i, int j) {
        int n = mat.size();
        return i >= 0 && j >= 0 && i < n && j < n;
    }
};
```

- **Time Complexity:** $O(n^2 \log n)$
- **Space Complexity:** $O(n^2)$

#### 핵심 아이디어

내 코드와 알고리즘은 동일하다. 차이점은 visited 배열을 별도로 두지 않고 **grid 값을 -1로 덮어써서 방문 처리**한다는 것이다. BFS 거리 계산 단계에서 이미 `grid[i][j] != -1`이면 거리값이 담긴 셀이므로, 방문 후 `-1`로 재설정하면 중복 방문을 막을 수 있다. 공간 최적화가 이루어진다.

### 풀이 비교

| 항목 | 내 코드 | Approach 1 (이진 탐색) | Approach 2 (Greedy) |
|------|---------|----------------------|---------------------|
| 시간 복잡도 | $O(n^2 \log n)$ | $O(n^2 \log n)$ | $O(n^2 \log n)$ |
| 공간 복잡도 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ |
| 2단계 알고리즘 | Dijkstra형 PQ | 이진 탐색 + BFS | Dijkstra형 PQ |
| visited 처리 | 별도 `visited` 배열 | 별도 `visited` 배열 | grid 값 -1 덮어쓰기 |
| 입력 grid 수정 | 수정하지 않음 | 수정함 | 수정함 |

세 풀이 모두 시간/공간 복잡도가 동일하다. Approach 2는 visited 배열을 생략해 공간 상수를 줄이지만 원본 grid를 변경한다는 트레이드오프가 있다. 내 코드는 원본 grid를 보존하면서 동일한 성능을 낸다.
