## 문제 설명

`a`, `b`, `c` 세 문자로만 구성된 문자열 `s`가 주어진다.  
`a`, `b`, `c`를 <strong>각각 하나 이상</strong> 포함하는 부분 문자열(substring)의 개수를 반환하라.

### 핵심 관찰

- 윈도우 `[f, b]`가 세 문자를 모두 포함하면, `[f, b]`, `[f, b+1]`, ..., `[f, length-1]` 모두 유효하다.
- 따라서 조건을 만족하는 순간 <strong>오른쪽으로 연장되는 모든 부분 문자열</strong>을 한 번에 셀 수 있다.

<br/>

## 내 코드 분석

```cpp
class Solution {
public:
    int numberOfSubstrings(string s) {
        vector<int> chk(3, 0);

        int length = s.size();
        int f{}, b{};
        int answer{};

        for (; b < length; ++b) {
            chk[s[b] - 'a']++;
            while (chk[0] && chk[1] && chk[2]) {
                answer += (length - b);
                chk[s[f] - 'a']--;
                ++f;
            }
        }

        return answer;
    }
};
```

- <strong>Time Complexity:</strong> $O(n)$
- <strong>Space Complexity:</strong> $O(1)$ (`chk`는 크기 3 고정)

### 동작 원리

`b`(오른쪽 포인터)를 확장하며 `chk`에 빈도를 누적한다.

세 문자가 모두 존재하면(`chk[0] && chk[1] && chk[2]`):
- 현재 왼쪽 끝이 `f`이고 오른쪽 끝이 `b`인 윈도우는 조건을 만족한다.
- 오른쪽을 `b`, `b+1`, ..., `length-1`까지 늘려도 여전히 유효하므로 `answer += (length - b)`.
- 왼쪽을 한 칸 줄여(`f++`) 더 작은 윈도우로도 조건이 성립하는지 계속 확인한다.

이중 루프처럼 보이지만 `f`와 `b` 모두 최대 `n`번만 이동하므로 $O(n)$이다.

<br/>

## 최적 풀이

### Approach 1: Sliding Window (명시적 헬퍼 함수)

내 코드와 동일한 알고리즘이지만, 포인터 이름을 `left`/`right`로 명확히 하고 조건 검사를 헬퍼 함수로 분리했다.

```cpp
class Solution {
public:
    int numberOfSubstrings(string s) {
        int len = s.length();
        int left = 0, right = 0;
        vector<int> freq(3, 0);
        int total = 0;

        while (right < len) {
            freq[s[right] - 'a']++;

            while (hasAllChars(freq)) {
                total += len - right;
                freq[s[left] - 'a']--;
                left++;
            }

            right++;
        }

        return total;
    }

private:
    bool hasAllChars(vector<int>& freq) {
        return freq[0] > 0 && freq[1] > 0 && freq[2] > 0;
    }
};
```

- <strong>Time Complexity:</strong> $O(n)$
- <strong>Space Complexity:</strong> $O(1)$

내 코드와 동작이 완전히 동일하며, 가독성 차이만 있다.

### Approach 2: Last Position Tracking

각 문자의 <strong>마지막 등장 위치</strong>만 추적하는 방식이다. 슬라이딩 윈도우 없이 단일 루프로 해결한다.

```cpp
class Solution {
public:
    int numberOfSubstrings(string s) {
        int len = s.length();
        vector<int> lastPos = {-1, -1, -1};
        int total = 0;

        for (int pos = 0; pos < len; pos++) {
            lastPos[s[pos] - 'a'] = pos;
            total += 1 + min({lastPos[0], lastPos[1], lastPos[2]});
        }

        return total;
    }
};
```

- <strong>Time Complexity:</strong> $O(n)$
- <strong>Space Complexity:</strong> $O(1)$

#### 핵심 아이디어

`pos` 위치에서 끝나는 유효한 부분 문자열의 수를 구한다.

세 문자의 마지막 위치 중 <strong>가장 작은 값</strong>을 `m = min(lastPos[0], lastPos[1], lastPos[2])`라 하면:
- 시작점이 `0`, `1`, ..., `m` 인 부분 문자열 `[start, pos]`는 모두 유효하다.
- 시작점이 `m+1` 이상이면 세 문자 중 하나가 빠진다.
- 따라서 `pos`에서 끝나는 유효한 부분 문자열 수 = `m + 1`.

`lastPos`가 초기값 `-1`이면 `m = -1`이 되어 `1 + (-1) = 0`이 더해진다. 세 문자가 아직 모두 등장하지 않은 경우를 자연스럽게 처리하는 트릭이다.

#### 예시

`s = "abcabc"`

| pos | s[pos] | lastPos | min | += |
|-----|--------|---------|-----|----|
| 0 | a | [0,-1,-1] | -1 | 0 |
| 1 | b | [0,1,-1] | -1 | 0 |
| 2 | c | [0,1,2] | 0 | 1 |
| 3 | a | [3,1,2] | 1 | 2 |
| 4 | b | [3,4,2] | 2 | 3 |
| 5 | c | [3,4,5] | 3 | 4 |

합계 = <strong>10</strong>
