[문제 링크](https://leetcode.com/problems/remove-covered-intervals/)

<br/>

## 문제 설명

정수 쌍으로 이루어진 `intervals` 배열이 주어진다. `intervals[i] = [li, ri]`는 구간 $[l_i, r_i)$를 나타낸다.

구간 $[a, b)$가 구간 $[c, d)$에 **covered** 된다고 함은 $c \le a$ 이고 $b \le d$인 경우다.

covered 되지 않는 구간의 수를 반환하라.

### 핵심 관찰

- $n \le 1000$이므로 $O(n^2)$ 브루트포스로도 충분히 통과할 수 있다.
- 정렬을 활용하면 $O(n \log n)$의 그리디 풀이도 가능하다: 시작점 기준 오름차순, 시작점이 같으면 끝점 기준 내림차순으로 정렬한 뒤 최대 끝점을 추적하며 단일 패스로 처리한다.

<br/>

## 내 아이디어

$n \le 1000$이므로 $O(n^2)$ 브루트포스로 풀기로 했다.

- 2중 for문으로 모든 구간 쌍 $(i, j)$를 확인한다.
- 구간 $i$가 구간 $j$에 covered 되면 `chk[i] = 1`로 표시하고 안쪽 루프를 즉시 break한다.
- 마지막에 `chk[i] == 0`인 구간만 카운팅한다.

<br/>

## 내 아이디어로 구현된 코드

```cpp
class Solution {
public:
    int removeCoveredIntervals(vector<vector<int>>& intervals) {
        int n = intervals.size();
        vector<int> chk(n, 0);

        for (int i = 0; i < n; i++) {
            int a = intervals[i][0];
            int b = intervals[i][1];

            for (int j = 0; j < n; j++) {
                if (i == j) continue;

                int c = intervals[j][0];
                int d = intervals[j][1];

                if (c <= a && b <= d) {
                    chk[i] = 1;
                    break;
                }
            }
        }

        int answer = 0;
        for (int i = 0; i < n; i++) {
            if (chk[i] == 0) answer++;
        }

        return answer;
    }
};
```

- **Time Complexity:** $O(n^2)$
- **Space Complexity:** $O(n)$

### 동작 원리

각 구간 $i$에 대해 자신을 cover하는 다른 구간 $j$가 존재하는지 전수 탐색한다. cover 조건은 $c \le a$ 이고 $b \le d$이다. 발견 즉시 break해 불필요한 추가 탐색을 줄인다. 탐색이 끝난 후 `chk[i] == 0`인 구간들만 세면 covered되지 않은 구간의 수를 얻는다.

$n \le 1000$이므로 최악의 경우 약 $10^6$번 비교가 일어나며 제한 시간 내에 충분히 통과한다.

<br/>

## 풀이 1: 정렬 + 그리디

```cpp
class Solution {
public:
    int removeCoveredIntervals(vector<vector<int>>& intervals) {
        sort(intervals.begin(), intervals.end(), [](const vector<int>& a, const vector<int>& b) {
            if (a[0] == b[0]) return a[1] > b[1];
            return a[0] < b[0];
        });

        int count = 0;
        int maxRight = 0;

        for (const auto& interval : intervals) {
            if (interval[1] > maxRight) {
                count++;
                maxRight = interval[1];
            }
        }

        return count;
    }
};
```

- **Time Complexity:** $O(n \log n)$
- **Space Complexity:** $O(\log n)$ (정렬 스택)

### 핵심 아이디어

**정렬 기준**: 시작점 오름차순, 시작점이 같으면 끝점 내림차순으로 정렬한다.

이 정렬 후에는 구간 $i$보다 앞에 있는 구간들의 시작점이 $\le intervals[i][0]$임이 보장된다. 따라서 "지금까지 본 구간들의 최대 끝점(`maxRight`)"이 현재 구간의 끝점 이상이라면, 현재 구간은 반드시 앞선 어떤 구간에 covered 된다.

- `interval[1] > maxRight`: 현재 끝점이 지금까지의 최대보다 크다 → covered되지 않음 → 카운트하고 `maxRight` 갱신
- `interval[1] <= maxRight`: 이미 더 큰 끝점을 가진 구간이 앞에 있고 시작점도 $\le$ 이므로 → covered됨 → 카운트하지 않음

시작점이 같을 때 끝점을 내림차순으로 정렬하는 이유: 같은 시작점을 가진 구간 중 끝점이 가장 큰 것이 먼저 처리되어 `maxRight`를 선점하면, 뒤따르는 더 짧은 구간들은 자연스럽게 covered로 처리된다.

#### 예시

`intervals = [[1,4],[3,6],[1,2]]`

정렬 후: `[[1,4],[1,2],[3,6]]` (시작점 같으면 끝점 내림차순)

| 구간 | interval[1] | maxRight | 처리 |
|------|-------------|----------|------|
| [1,4] | 4 | 0 → 4 | 카운트 (4 > 0) |
| [1,2] | 2 | 4 | 스킵 (2 ≤ 4) |
| [3,6] | 6 | 4 → 6 | 카운트 (6 > 4) |

결과: **2**

## 풀이 비교

| 항목 | 내 코드 | 풀이 1 (정렬 + 그리디) |
|------|---------|----------------------|
| 시간 복잡도 | $O(n^2)$ | $O(n \log n)$ |
| 공간 복잡도 | $O(n)$ | $O(\log n)$ |
| 핵심 기법 | 전수 탐색 | 정렬 후 단일 패스 |
| 구현 난이도 | 쉬움 | 정렬 기준 설계 필요 |

$n \le 1000$인 이 문제에서는 두 풀이 모두 통과하지만, 풀이 1이 시간/공간 모두 더 효율적이다. 특히 $n$이 $10^5$ 수준으로 커진다면 풀이 1이 필수적이다.
