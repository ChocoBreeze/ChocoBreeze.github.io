## 문제 설명

정수 배열 `arr`가 주어진다. 다음 두 가지 연산을 **임의의 횟수**만큼 수행할 수 있다.

1. 임의의 원소를 선택하여 **감소**시킨다 (단, 값이 1 이상이어야 한다).
2. 배열 원소를 **임의의 순서로 재배열**한다.

연산 후 배열이 아래 조건을 만족해야 한다.

- `arr[0] == 1`
- 인접한 원소의 차이: `|arr[i] - arr[i-1]| <= 1`

이 조건을 만족하면서 배열의 **최댓값을 최대화**하는 것이 목표다.

### 핵심 관찰

- 원소를 **증가**시키는 연산은 없다. 따라서 큰 값을 작게 만들 수는 있지만 반대는 불가능하다.
- `arr[0] = 1`이고 인접 차이가 최대 1이므로, n개 원소로 만들 수 있는 최대 수열은 `1, 2, 3, ..., n`이다.
- 즉, **최댓값의 상한은 `n`**이다.

<br/>

## 내 코드 분석

```cpp
class Solution {
public:
    int maximumElementAfterDecrementingAndRearranging(vector<int>& arr) {
        sort(begin(arr), end(arr));

        int size = static_cast<int>(arr.size());
        
        arr[0] = 1;
        for(int i{1};i<size;++i) {
            int diff = arr[i] - arr[i-1];
            if(diff > 1) {
                arr[i] = arr[i-1] + 1;
            }
        }

        return arr.back();
    }
};
```

- **Time Complexity:** $O(n \log n)$
- **Space Complexity:** $O(\log n)$ (정렬 스택)

### 동작 원리

1. **정렬**: 오름차순으로 정렬하면 각 위치에서 가능한 최댓값을 탐욕적으로 계산할 수 있다.
2. **`arr[0] = 1`**: 첫 번째 원소를 강제로 1로 설정한다.
3. **그리디 조정**: `i`번 원소와 `i-1`번 원소의 차이가 1을 초과하면, `arr[i] = arr[i-1] + 1`로 축소한다.
   - `diff == 0` 또는 `diff == 1`이면 현재 값 유지 → 이미 조건 만족
   - `diff > 1`이면 최대한 늘린 값(`arr[i-1] + 1`)으로 감소

이 로직은 `arr[i] = min(arr[i], arr[i-1] + 1)`과 동일하다. 정렬된 배열이므로 `arr[i] >= arr[i-1]`이 항상 성립하여 음수 diff는 발생하지 않는다.

### 정확성

배열이 정렬되어 있기 때문에, 각 위치 `i`에서 달성 가능한 최댓값은 `arr[i-1] + 1`이다. 이를 넘는 값은 줄여야 하고, 이미 그 이하라면 그대로 두면 된다. 탐욕적으로 각 위치의 값을 최대화하면 전체 최댓값도 최대가 된다.

<br/>

## 최적 풀이

정렬 없이 **카운팅**만으로 $O(n)$에 해결하는 방법이다.

```cpp
class Solution {
public:
    int maximumElementAfterDecrementingAndRearranging(vector<int>& arr) {
        int n = arr.size();
        vector<int> counts(n + 1, 0);

        for (int num : arr) {
            counts[min(num, n)]++;
        }

        int ans = 1;
        for (int num = 2; num <= n; num++) {
            ans = min(ans + counts[num], num);
        }

        return ans;
    }
};
```

- **Time Complexity:** $O(n)$
- **Space Complexity:** $O(n)$

### 핵심 아이디어

최댓값의 상한은 $n$이다. 따라서 $n$을 초과하는 원소는 $n$과 동일하게 취급해도 무방하다.

**counts 배열 구성**

```
counts[min(num, n)]++
```

각 원소를 `min(num, n)`으로 클램핑한 뒤 카운팅한다. $n$보다 큰 원소들은 모두 `counts[n]`에 누적된다.

**그리디 탐색**

```
ans = 1   // arr[0] = 1 고정
for num = 2 to n:
    ans = min(ans + counts[num], num)
```

- `ans`: 현재까지 연속으로 채울 수 있는 최대값
- `counts[num]`: `num` 이상인 원소 중 이 위치에 기여할 수 있는 수
- `ans + counts[num]`: 새 원소를 추가해 늘릴 수 있는 최대값
- `min(..., num)`: 위치 `num`에서의 절대적 상한 (1, 2, ..., num 수열의 끝이 `num`이므로)

### 예시

`arr = [2, 2, 1, 2, 1]`, n = 5

| num | counts[num] | ans 계산 | ans |
|-----|-------------|----------|-----|
| (초기) | - | - | 1 |
| 2 | 3 | min(1+3, 2) = 2 | 2 |
| 3 | 0 | min(2+0, 3) = 2 | 2 |
| 4 | 0 | min(2+0, 4) = 2 | 2 |
| 5 | 2 | min(2+2, 5) = 4 | 4 |

→ 반환값: **4**

### 내 코드와의 비교

| 항목 | 내 코드 (정렬) | 최적 풀이 (카운팅) |
|------|--------------|-----------------|
| 시간 복잡도 | $O(n \log n)$ | $O(n)$ |
| 공간 복잡도 | $O(\log n)$ | $O(n)$ |
| 정렬 필요 | O | X |
| 원본 배열 수정 | O | X |

정렬을 완전히 생략하고, 값의 분포만 파악해 그리디하게 답을 구한다. 공간을 $O(n)$ 추가로 사용하는 대신 시간을 $O(n)$으로 줄인 트레이드오프다.
