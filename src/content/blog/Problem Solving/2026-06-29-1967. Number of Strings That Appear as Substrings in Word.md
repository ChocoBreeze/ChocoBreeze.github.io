## 문제 설명

문자열 배열 `patterns`와 문자열 `word`가 주어진다.

- `patterns[i]`가 `word`의 <strong>부분 문자열(substring)</strong> 로 등장하는지 확인한다.
- 부분 문자열로 등장하는 `patterns[i]`의 <strong>개수</strong>를 반환한다.

### 핵심 관찰

- 각 패턴이 `word` 안에 등장하는지를 독립적으로 판단하면 된다.
- 부분 문자열 탐색 알고리즘(Brute Force, KMP 등)을 각 패턴에 적용하면 된다.
- C++23의 `std::string::contains()`를 사용하면 표준 라이브러리에 탐색을 위임할 수 있다.

<br/>

## 내 코드 분석

```cpp
class Solution {
public:
    int numOfStrings(vector<string>& patterns, string word) {
        int cnt{};
        for(string& pattern : patterns) {
            if(word.contains(pattern)) {
                ++cnt;
            }
        }
        return cnt;
    }
};
```

- <strong>Time Complexity:</strong> $O(\sum m_i \times n)$ — 표준 라이브러리의 내부 구현에 의존하며, 최악의 경우 각 패턴 길이 $m_i$와 `word` 길이 $n$의 곱에 비례한다.
- <strong>Space Complexity:</strong> $O(1)$

### 동작 원리

C++23에서 추가된 `std::string::contains()`를 사용한다. 각 패턴을 순회하며 `word`가 해당 패턴을 포함하면 카운터를 증가시킨다. 탐색 로직 자체는 표준 라이브러리에 위임하므로 코드가 매우 간결하다.

<br/>

## 최적 풀이

### Approach 1: Brute Force Matching

```cpp
class Solution {
public:
    int numOfStrings(vector<string>& patterns, string word) {
        auto check = [](const string& pattern, const string& word) -> bool {
            int m = pattern.size();
            int n = word.size();
            for (int i = 0; i + m <= n; ++i) {
                bool flag = true;
                for (int j = 0; j < m; ++j) {
                    if (word[i + j] != pattern[j]) {
                        flag = false;
                        break;
                    }
                }
                if (flag) {
                    return true;
                }
            }
            return false;
        };

        int res = 0;
        for (const string& pattern : patterns) {
            res += check(pattern, word);
        }
        return res;
    }
};
```

- <strong>Time Complexity:</strong> $O(\sum m_i \times n)$ — 각 패턴마다 `word`의 모든 시작 위치를 시도한다.
- <strong>Space Complexity:</strong> $O(1)$

#### 동작 원리

`word`의 각 인덱스 $i$를 패턴의 시작점으로 가정하고, 길이 $m$인 부분 문자열과 패턴을 문자 단위로 비교한다. 중간에 불일치가 발생하면 즉시 중단(early exit)한다.

### Approach 2: KMP (Knuth-Morris-Pratt)

```cpp
class Solution {
public:
    int numOfStrings(vector<string>& patterns, string word) {
        auto check = [](const string& pattern, const string& word) -> bool {
            int m = pattern.size();
            int n = word.size();
            // generate the prefix array of the pattern
            vector<int> pi(m);
            for (int i = 1, j = 0; i < m; i++) {
                while (j > 0 && pattern[i] != pattern[j]) {
                    j = pi[j - 1];
                }
                if (pattern[i] == pattern[j]) {
                    ++j;
                }
                pi[i] = j;
            }
            // using prefix arrays for matching
            for (int i = 0, j = 0; i < n; i++) {
                while (j > 0 && word[i] != pattern[j]) {
                    j = pi[j - 1];
                }
                if (word[i] == pattern[j]) {
                    ++j;
                }
                if (j == m) {
                    return true;
                }
            }
            return false;
        };

        int res = 0;
        for (const string& pattern : patterns) {
            res += check(pattern, word);
        }
        return res;
    }
};
```

- <strong>Time Complexity:</strong> $O(\sum (m_i + n))$ — 각 패턴에 대해 prefix 배열 구성이 $O(m_i)$, 탐색이 $O(n)$이다.
- <strong>Space Complexity:</strong> $O(m)$ — 현재 패턴의 prefix 배열을 저장한다.

#### 동작 원리

KMP는 불일치 발생 시 이미 검사한 정보를 버리지 않고, <strong>prefix 함수(failure function)</strong> `pi`를 이용해 다음 비교 위치로 점프한다.

`pi[i]`는 `pattern[0..i]`의 <strong>접두사이면서 동시에 접미사인 가장 긴 문자열의 길이</strong>를 저장한다.

예를 들어 `pattern = "abab"`이라면:

| i | pi[i] | 의미 |
|---|-------|------|
| 0 | 0 | 없음 |
| 1 | 0 | 없음 |
| 2 | 1 | "a" |
| 3 | 2 | "ab" |

탐색 중 불일치가 발생하면 `j = pi[j-1]`로 되돌아가 비교를 재개한다. 이로써 `word`의 각 문자를 최대 1회만 방문한다.

### 풀이 비교

| 항목 | 내 코드 | Approach 1 (Brute Force) | Approach 2 (KMP) |
|------|---------|--------------------------|-----------------|
| 시간 복잡도 | $O(\sum m_i \times n)$ (구현 의존) | $O(\sum m_i \times n)$ | $O(\sum (m_i + n))$ |
| 공간 복잡도 | $O(1)$ | $O(1)$ | $O(m)$ |
| 구현 복잡도 | 매우 낮음 | 낮음 | 높음 |
| C++ 버전 | C++23 이상 필요 | 모든 버전 | 모든 버전 |

이 문제의 제약 조건(`patterns` 최대 100개, `word` 최대 길이 100)은 매우 작기 때문에 세 풀이 모두 실질적인 성능 차이는 없다. `std::string::contains()`는 표준 라이브러리 최적화를 활용할 수 있어 실용적으로 가장 간결하다. KMP는 패턴과 `word`가 모두 길어지는 대규모 입력에서 이론적 우위를 갖는다.
