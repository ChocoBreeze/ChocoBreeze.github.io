[문제 링크](https://leetcode.com/problems/find-a-safe-walk-through-a-grid/)

<br/>

## 문제 설명

`m × n` 이진 그리드 `grid`와 정수 `health`가 주어진다.

- `grid[i][j] == 0`이면 안전한 칸, `1`이면 위험한 칸이다.
- 위험한 칸을 밟으면 체력이 1 감소한다. 안전한 칸은 체력이 줄지 않는다.
- 시작점은 `(0, 0)`, 도착점은 `(m-1, n-1)`이며 상하좌우로 이동 가능하다.
- **체력은 항상 양수(> 0)** 를 유지해야 한다. 0이 되는 순간 사망한다.

`(0, 0)`에서 `(m-1, n-1)`까지 체력을 양수로 유지하며 이동할 수 있으면 `true`, 아니면 `false`를 반환하라.

### 핵심 관찰

- 각 칸의 이동 비용이 0 또는 1이므로, **소모 체력 최소 경로**를 구하면 된다.
- 최소 소모 체력이 `health` 미만이면 도달 가능하다.
- 반대로, "남은 체력이 가장 많은 경로"를 우선 탐색해도 동일한 결론에 도달한다.

<br/>

## 내 아이디어

Priority Queue(최대 힙)를 사용해 **남은 체력이 가장 많은 상태를 우선 탐색**한다.

- 큐에 `{남은 체력, x, y}`를 넣고 내림차순으로 꺼낸다.
- `best[x][y]`에 해당 칸에 도달했을 때의 최대 체력을 기록한다.
- 같은 칸이라도 더 많은 체력으로 도달하면 다시 탐색해야 한다.
- 큐에서 꺼낼 때 `best[x][y]`보다 낮은 체력이면 스킵한다 (이미 더 좋은 경로로 처리됨).
- `(m-1, n-1)`에 처음 도달하면 즉시 `true`를 반환한다. 큐를 모두 비워도 도달 못하면 `false`.

<br/>

## 내 아이디어로 구현된 코드

```cpp
class Solution {
public:
    bool findSafeWalk(vector<vector<int>>& grid, int health) {
        int m = grid.size();
        int n = grid[0].size();

        // 시작 칸 비용도 반영
        int startHealth = health - grid[0][0];
        if (startHealth <= 0) return false;

        // best[x][y] = (x, y)에 도착했을 때 남아있던 최대 체력
        vector<vector<int>> best(m, vector<int>(n, -1));
        best[0][0] = startHealth;

        // {남은 체력, x, y}
        priority_queue<tuple<int, int, int>> pq;
        pq.push({startHealth, 0, 0});

        int dx[4] = {-1, 1, 0, 0};
        int dy[4] = {0, 0, -1, 1};

        while (!pq.empty()) {
            auto [curHealth, x, y] = pq.top();
            pq.pop();

            // 이미 더 좋은 체력으로 처리된 상태면 스킵
            if (curHealth < best[x][y]) continue;

            // 도착했고 체력이 양수면 성공
            if (x == m - 1 && y == n - 1) {
                return true;
            }

            for (int dir = 0; dir < 4; dir++) {
                int nx = x + dx[dir];
                int ny = y + dy[dir];

                if (nx < 0 || nx >= m || ny < 0 || ny >= n) continue;

                int nextHealth = curHealth - grid[nx][ny];

                // 체력은 항상 positive여야 함
                if (nextHealth <= 0) continue;

                // 더 좋은 체력으로 도착할 수 있을 때만 갱신
                if (nextHealth > best[nx][ny]) {
                    best[nx][ny] = nextHealth;
                    pq.push({nextHealth, nx, ny});
                }
            }
        }

        return false;
    }
};
```

- **Time Complexity:** $O(mn \log(mn))$ — 각 칸이 큐에 여러 번 들어갈 수 있으나, `best` 배열로 가지치기되어 $O(mn \log(mn))$에 수렴한다.
- **Space Complexity:** $O(mn)$

### 동작 원리

**시작 처리:** `(0, 0)` 칸 자체도 비용이 발생할 수 있으므로, `startHealth = health - grid[0][0]`으로 초기 체력을 계산한다. 이 값이 0 이하면 즉시 `false`를 반환한다.

**최대 힙 탐색:** 항상 남은 체력이 가장 많은 경로를 먼저 꺼낸다. `(m-1, n-1)`에 처음 도달하는 경로가 바로 최적 경로이므로, 도달 즉시 `true`를 반환할 수 있다.

**중복 처리:** `best[x][y]`를 관리해 이미 더 좋은 체력으로 방문한 칸은 재처리하지 않는다. 체력이 `best[x][y]`보다 낮은 상태로 큐에서 꺼내지면 스킵한다.

## 풀이 1: Dijkstra (누적 비용 최소화)

```cpp
class Solution {
public:
    bool findSafeWalk(vector<vector<int>>& grid, int health) {
        int m = grid.size(), n = grid[0].size();
        vector<vector<int>> dis(m, vector<int>(n, -1));
        int dirs[4][2] = {{0, 1}, {1, 0}, {-1, 0}, {0, -1}};

        priority_queue<tuple<int, int, int>, vector<tuple<int, int, int>>,
                       greater<>>
            pq;
        pq.emplace(grid[0][0], 0, 0);
        while (!pq.empty()) {
            auto [val, cx, cy] = pq.top();
            pq.pop();
            if (dis[cx][cy] >= 0) {
                continue;
            }
            dis[cx][cy] = val;
            for (int k = 0; k < 4; k++) {
                int nx = cx + dirs[k][0];
                int ny = cy + dirs[k][1];
                if (nx < 0 || ny < 0 || nx >= m || ny >= n ||
                    dis[nx][ny] >= 0) {
                    continue;
                }
                pq.emplace(val + grid[nx][ny], nx, ny);
            }
        }

        return dis[m - 1][n - 1] < health;
    }
};
```

- **Time Complexity:** $O(mn \log(mn))$
- **Space Complexity:** $O(mn)$

### 핵심 아이디어

발상을 뒤집어, **남은 체력 최대화** 대신 **누적 소모 체력 최소화**로 접근한다.

- `{누적 비용, x, y}`를 최소 힙으로 관리한다.
- 어떤 칸을 큐에서 처음 꺼낸 시점의 비용이 곧 그 칸까지의 최단 비용이다(Dijkstra 보장).
- `dis[cx][cy] >= 0`인 칸은 이미 확정되었으므로 스킵한다.
- 최종적으로 `dis[m-1][n-1] < health`이면 도달 가능하다.

내 코드가 "남은 체력 최대화"(최대 힙)를 추적하는 것과 달리, 이 풀이는 "소모 비용 최소화"(최소 힙)를 추적한다. 두 접근은 수학적으로 동치지만, Dijkstra 방식은 각 칸이 큐에서 **단 한 번만** 확정되므로 큐 연산이 더 적다.

## 풀이 2: 0-1 BFS

```cpp
class Solution {
    static constexpr int DIRS[4][2] = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};

public:
    bool findSafeWalk(vector<vector<int>>& grid, int health) {
        int m = grid.size(), n = grid[0].size();
        vector<vector<int>> dis(m, vector<int>(n, INT_MAX));

        deque<pair<int, int>> q;
        q.emplace_front(0, 0);
        dis[0][0] = grid[0][0];

        while (!q.empty()) {
            auto [cx, cy] = q.front();
            q.pop_front();
            if (cx == m - 1 && cy == n - 1) {
                return true;
            }

            for (auto& [dx, dy] : DIRS) {
                int nx = cx + dx, ny = cy + dy;
                if (nx < 0 || ny < 0 || nx >= m || ny >= n) {
                    continue;
                }
                int cost = dis[cx][cy] + grid[nx][ny];
                // pruning: the new distance does not meet health requirements
                if (cost >= health) {
                    continue;
                }
                if (cost < dis[nx][ny]) {
                    dis[nx][ny] = cost;
                    if (grid[nx][ny] == 0) {
                        q.emplace_front(nx, ny);
                    } else {
                        q.emplace_back(nx, ny);
                    }
                }
            }
        }

        return false;
    }
};
```

- **Time Complexity:** $O(mn)$ — 0-1 가중치 그래프에서 0-1 BFS는 선형 시간
- **Space Complexity:** $O(mn)$

### 핵심 아이디어

이 문제의 간선 가중치는 항목 0 또는 1뿐이다. 이 특성을 이용한 **0-1 BFS**로 Priority Queue 없이 $O(mn)$에 최단 경로를 구할 수 있다.

- 비용이 0인 이동(안전 칸)은 `deque`의 **앞**에 삽입한다.
- 비용이 1인 이동(위험 칸)은 `deque`의 **뒤**에 삽입한다.
- 앞에서 꺼낼 때 항상 비용이 낮은 상태가 먼저 처리되어, Priority Queue 없이도 Dijkstra와 동일한 순서를 보장한다.
- `cost >= health`인 경우를 미리 가지치기해 불필요한 탐색을 제거한다.

## 풀이 비교

| 항목 | 내 코드 (최대 힙) | 풀이 1 (Dijkstra) | 풀이 2 (0-1 BFS) |
|------|-----------------|-------------------|-----------------|
| 시간 복잡도 | $O(mn \log(mn))$ | $O(mn \log(mn))$ | $O(mn)$ |
| 공간 복잡도 | $O(mn)$ | $O(mn)$ | $O(mn)$ |
| 핵심 기법 | 남은 체력 최대화 | 누적 비용 최소화 | 0-1 가중치 특성 활용 |
| 자료구조 | 최대 힙 | 최소 힙 | deque |
| 칸당 처리 횟수 | 여러 번 가능 | 단 한 번 | 단 한 번 |

가중치가 0과 1뿐이라면 **0-1 BFS가 이론적으로 가장 빠르다**. 내 코드와 풀이 1은 Priority Queue를 사용해 $O(\log)$ 오버헤드가 있지만, 이 문제의 제약 조건($m, n \le 50$)에서는 세 풀이 모두 실질적인 차이가 없다.
