## 문제 설명

정수 `n`이 주어진다.

- `n`의 자릿수 중 **0이 아닌 자릿수만** 순서대로 이어 붙여 새로운 정수 `x`를 만든다.
- `n`을 이루는 **모든 자릿수의 합** `sum`을 구한다. (0인 자릿수는 더해도 값에 영향이 없으므로 사실상 0이 아닌 자릿수의 합과 같다.)
- 최종적으로 `x * sum`을 반환해야 한다.

핵심 관찰은 두 가지다.

- `n`의 첫 자릿수는 항상 0이 아니므로(선행 0이 없으므로), 0이 아닌 자릿수를 이어 붙여도 `x`는 최소 1자리 이상의 유효한 수가 된다.
- 자릿수를 왼쪽에서 오른쪽으로 읽든, 오른쪽에서 왼쪽으로 읽든 상관없이 0인 자릿수만 건너뛰면서 순서만 유지하면 같은 `x`를 만들 수 있다.

<br/>

## 내 아이디어

그냥 구현하는 방식으로 접근했다.

- `n`을 자릿수 단위로 분리하면서 동시에 전체 자릿수 합(`sum`)을 누적한다.
- 이때 0인 자릿수는 제외하고 0이 아닌 자릿수만 별도의 벡터(`rev`)에 저장해둔다. `n`을 `% 10`, `/ 10` 방식으로 분리하면 자연히 **오른쪽(1의 자리)부터 왼쪽 방향**으로 자릿수를 얻게 되므로, `rev`에는 원래 순서가 뒤집힌 채로 자릿수가 쌓인다.
- 이후 `rev`를 그대로 순회하면서 `holder`(자릿값, 1 → 10 → 100 …)를 곱해 누적하면, 뒤집힌 순서를 다시 원래 순서로 복원하면서 숫자 `x`를 재구성할 수 있다.
- 만약 0이 아닌 자릿수가 하나도 없다면(`rev`가 비어 있다면) `x`는 존재하지 않으므로 0을 반환한다.
- 마지막에 `x * sum`을 반환한다.

<br/>

## 내 아이디어로 구현된 코드

```cpp
class Solution {
public:
    long long sumAndMultiply(int n) {
        vector<int> rev;
        int sum{};
        while(n) {
            if(n%10) {
                int tmp = n % 10;
                rev.push_back(tmp);
                sum += tmp;
            }
            n /= 10;
        }
        if(rev.empty()) {
            return 0;
        }

        long long ret{}, holder=1;
        for(int i : rev) {
            ret += holder * i;
            holder *= 10;
        }

        return ret * sum;
    }
};
```

- **Time Complexity:** $O(\log n)$
- **Space Complexity:** $O(\log n)$

### 동작 원리

1. `n`이 0이 될 때까지 `n % 10`으로 마지막 자릿수를 꺼내고 `n /= 10`으로 한 자리씩 줄여나간다. 이 과정에서 자릿수는 항상 **1의 자리 → 10의 자리 → 100의 자리 …** 순서, 즉 원래 숫자와는 반대 방향으로 나온다.
2. 자릿수가 0이 아닐 때만 `rev`에 저장하고 `sum`에도 더한다. (0인 자릿수는 `sum`에 더해도 0이라 영향이 없으므로 애초에 조건 없이 더해도 결과는 같지만, 여기서는 조건 안에서 함께 처리했다.)
3. 자릿수를 모두 꺼낸 뒤 `rev`에는 자릿수가 뒤집힌 순서로 들어있다. 이를 앞에서부터 순회하면서 `holder`(현재까지 처리한 자릿수의 자릿값)를 곱해 더해주면, 뒤집힌 순서가 다시 정순으로 복원되면서 숫자가 재구성된다.
4. 0이 아닌 자릿수가 하나도 없는 경우(이론상 `n`이 항상 양수라 발생하지 않지만) 방어적으로 0을 반환한다.
5. 최종적으로 복원한 `x`(`ret`)와 자릿수 합 `sum`을 곱해 반환한다.

<br/>

## 풀이 1: Traverse from Left to Right

```cpp
class Solution {
public:
    long long sumAndMultiply(int n) {
        long long x = 0;
        long long sum = 0;
        string s = to_string(n);
        for (char c : s) {
            int d = c - '0';
            sum += d;
            if (d > 0) {
                x = x * 10 + d;
            }
        }
        return x * sum;
    }
};
```

- **Time Complexity:** $O(\log n)$
- **Space Complexity:** $O(\log n)$ (문자열 `s` 저장 공간)

### 핵심 아이디어

`n`을 `to_string`으로 문자열로 변환하면 자릿수가 **왼쪽(가장 높은 자리)부터 오른쪽(1의 자리)** 순서로 그대로 나열된다.

이 순서를 그대로 순회하면서 0이 아닌 자릿수를 만날 때마다 `x = x * 10 + d`로 뒤에 이어 붙이면, 별도의 뒤집기 과정 없이 바로 원하는 순서의 숫자를 만들 수 있다.

내 코드가 자릿수를 거꾸로 얻은 뒤 다시 복원하는 두 단계를 거치는 것과 달리, 이 풀이는 문자열 변환을 통해 자릿수를 원래 순서 그대로 얻기 때문에 복원 과정이 필요 없어 더 간결하다. 다만 `to_string` 호출과 문자열 저장에 따른 약간의 오버헤드가 있다.

<br/>

## 풀이 2: Traverse from Right to Left

```cpp
class Solution {
public:
    long long sumAndMultiply(int n) {
        long long x = 0, sum = 0, pow10 = 1;
        while (n > 0) {
            int d = n % 10;
            sum += d;
            if (d > 0) {
                x += d * pow10;
                pow10 *= 10;
            }
            n /= 10;
        }
        return x * sum;
    }
};
```

- **Time Complexity:** $O(\log n)$
- **Space Complexity:** $O(1)$

### 핵심 아이디어

내 코드와 마찬가지로 `n % 10`, `n /= 10`으로 1의 자리부터 순서대로 자릿수를 꺼낸다.

차이는 자릿수를 별도의 벡터에 저장한 뒤 다시 순회하며 복원하는 대신, **0이 아닌 자릿수를 만날 때마다 그 자리에서 바로 `pow10`(현재까지 만든 0이 아닌 자릿수 개수에 해당하는 자릿값)을 곱해 `x`에 더한다**는 점이다.

즉, 자릿수를 뒤에서부터 처리하지만 `pow10`을 0이 아닌 자릿수를 만날 때만 갱신함으로써, 벡터에 저장하지 않고도 한 번의 순회만으로 정순 숫자를 즉시 재구성한다. 이 덕분에 추가 자료구조 없이 $O(1)$ 공간만 사용한다.

## 풀이 비교

| 항목 | 내 코드 | 풀이 1 | 풀이 2 |
|------|---------|--------|--------|
| 시간 복잡도 | $O(\log n)$ | $O(\log n)$ | $O(\log n)$ |
| 공간 복잡도 | $O(\log n)$ (벡터 저장) | $O(\log n)$ (문자열 저장) | $O(1)$ |
| 핵심 기법 | 역순으로 저장 후 재순회하여 복원 | 문자열 변환으로 정순 그대로 순회 | 역순 순회 중 자릿값을 즉시 갱신해 한 번에 복원 |

세 풀이 모두 로직은 동일하지만, 별도의 저장 공간 없이 한 번의 순회로 끝내고 싶다면 풀이 2가 가장 효율적이고, 코드의 가독성을 우선한다면 문자열을 그대로 순회하는 풀이 1이 이해하기 쉽다.
